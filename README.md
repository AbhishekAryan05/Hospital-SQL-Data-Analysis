# 🏥 Hospital SQL Data Analysis

An end-to-end **SQL Data Analysis Project** built using **PostgreSQL** to analyze hospital operational data and generate meaningful business insights. This project demonstrates SQL skills including data aggregation, filtering, grouping, sorting, date calculations, and business analysis.

---

## 📸 Project Overview

This project analyzes hospital data to answer real-world business questions related to patient admissions, hospital performance, department workload, and medical expenses.

---

## 🎯 Project Objective

The objective of this project is to analyze hospital operational data using SQL and derive actionable business insights that can support better decision-making and resource planning.

---

## 📂 Dataset Information

The dataset contains the following fields:

- Hospital Name
- Location
- Department
- Doctors Count
- Patients Count
- Admission Date
- Discharge Date
- Medical Expenses

---

## 🛠️ SQL Concepts Used

- SELECT
- WHERE
- GROUP BY
- ORDER BY
- Aggregate Functions (SUM, AVG, COUNT)
- LIMIT
- Aliases (AS)
- Date Calculations
- Data Analysis Queries

---

## 📊 Business Questions Solved

- Calculate the total number of patients.
- Find the average number of doctors available in each hospital.
- Identify the top departments with the highest patient count.
- Calculate average daily medical expenses for each hospital.
- Find the hospital with the longest patient stay.
- Analyze department-wise patient distribution.
- Compare hospital performance using operational metrics.
- Evaluate medical expenses across hospitals.

---

## 💡 Key Business Insights

- Departments such as General Medicine and Emergency handle the highest patient volumes.
- Some hospitals have significantly higher doctor-to-patient ratios than others.
- Medical expenses vary considerably across hospitals.
- Longer hospital stays are associated with higher treatment costs.
- SQL analysis helps identify opportunities for better resource allocation and operational efficiency.
